Я работаю над проектом ассистента для лотерей, у меня есть общий репозиторий с backend на Go и frontend на TypeScript/React.

Твоя задача — ДОДЕЛАТЬ парсер данных с API Столото в go-backend и ПРИМЕНИТЬ эти данные на фронтенде.

Сделай всё по пунктам:

1. ОРИЕНТАЦИЯ В ПРОЕКТЕ
   - Открой файл go-backend/README.md.
   - Внимательно прочитай разделы, которые описывают:
     - какие именно данные по тиражам и играм Столото нам нужны;
     - какие эндпоинты Столото используются;
     - целевую структуру данных, которую должен возвращать backend.
   - Сформулируй для себя ЧЁТКИЙ контракт: какие поля и в каком формате backend должен отдавать фронту (id тиража, тип игры, дата, стоимость билета, суперприз, частота, winRate/winSize и т.п.).

2. ДОРАБОТКА ПАРСЕРА В GO-BACKEND
   - Найди текущую реализацию работы с API Столото в go-backend:
     - посмотри директории вида:
       - go-backend/internal/stoloto
       - go-backend/internal/services/stoloto
       - go-backend/internal/clients/stoloto
       - go-backend/pkg/stoloto
       - а также все файлы, где есть упоминания "Stoloto", "draws", "lotteries", "BestOf" и т.п.
   - Определи, где сейчас происходит:
     - HTTP-запрос к API Столото;
     - парсинг JSON-ответа;
     - маппинг этих данных в доменные структуры проекта.

   - Дальше:
     1) Сделай устойчивый HTTP-клиент:
        - аккуратно собери URL Столото (без хардкода магических строк по всему коду).
        - добавь таймауты и базовую обработку сетевых ошибок.
     2) Реализуй корректный парсинг JSON:
        - опиши Go-структуры под реальный ответ API Столото (с учётом README.md и примеров);
        - правильно проставь json-теги;
        - обработай вложенные объекты и массивы;
        - не теряй важные поля (дата тиража, номер, суперприз, цена билета, статус, тип лотереи и т.д.).
     3) Сделай преобразование в доменную модель:
        - создай или доработай структуры, которые будут возвращаться во внешнем API нашего backend (например, Lottery, Draw, StolotoGame, UniversalProps или аналоги, которые уже описаны в README.md);
        - приведи имена полей к понятному API-контракту (camelCase для JSON).
     4) Добавь логирование:
        - логируй ошибки запросов к Столото;
        - логируй случаи некорректного JSON/отсутствия обязательных полей.

   - Экспортируй конечную функцию, которая:
     - дергает API Столото;
     - парсит данные;
     - маппит в доменную структуру;
     - возвращает готовый список игр/тиражей в понятном виде.

3. HTTP-ЭНДПОИНТ ДЛЯ ФРОНТА
   - Найди в go-backend HTTP-сервер и роутер (например, main.go, internal/http, internal/handlers и т.п.).
   - Создай или доработай handler, который:
     - вызывается по понятному URL, например: GET /api/stoloto/draws или /api/stoloto/lotteries;
     - внутри использует доработанный парсер;
     - возвращает JSON с данными, которые нужны фронтенду (с тем контрактом, который ты сформулировал по README.md).
   - Обеспечь:
     - корректные коды ответа (200, 4xx, 5xx при ошибках);
     - нормальный Content-Type: application/json;
     - аккуратную обработку ошибок (человекочитаемое сообщение плюс лог на сервере).

   - После изменений запусти:
     - go vet ./...
     - go test ./...
     - и (если это возможно в окружении) основной бинарь для ручной проверки эндпоинта.

4. ИНТЕГРАЦИЯ С FRONTEND (TYPESCRIPT/REACT)
   - Перейди в frontend.
   - Найди модуль работы с API Столото и/или общим backend API, например:
     - src/lib/stolotoApi.ts
     - src/lib/api/stolotoApi.ts
     - src/lib/api/client.ts
     - любые файлы, где уже есть import stolotoApi или где фронт ожидает данные лотерей.
   - Создай или обнови функцию, которая:
     - обращается к новому backend-эндпоинту (например, /api/stoloto/draws);
     - типизируй ответ с помощью интерфейсов TypeScript (Lottery, Draw, StolotoGame, UniversalProps и т.п., в соответствии с тем контрактом, который реализовал backend);
     - корректно обрабатывай ошибки запроса.

   - Если в проекте используется React Query (TanStack Query):
     - добавь хук вида useStolotoDraws или подобный;
     - настрой ключи кэша;
     - обрабатывай loading / error / success.

5. ПРИМЕНЕНИЕ ДАННЫХ НА ФРОНТЕ
   - Найди компоненты, которые должны использовать данные Столото:
     - src/components/assistant/Assistant.tsx
     - src/components/assistant/ui/QuickRecommendations.tsx
     - src/pages/AnalyticsDashboard.tsx
     - src/components/lotteries, src/components/games и т.п.
   - Подключи туда новый хук/функцию получения данных:
     - отрисуй список игр/тиражей (название лотереи, номер тиража, дата, суперприз, цена билета и т.д.);
     - убери мок-данные, если они ещё используются;
     - следи за тем, чтобы типы (TypeScript) совпадали с ответом backend.

   - Важно:
     - Не ломай существующую архитектуру, аккуратно адаптируй её под реальные данные;
     - Сохрани текущий стиль кода и форматирование.

6. ПРОВЕРКА И ОБЪЯСНЕНИЕ
   - После всех правок:
     - Покажи полный список изменённых файлов и их содержимое.
     - Кратко опиши, ЧТО и ЗАЧЕМ ты поменял:
       - в go-backend (клиент Столото, парсер, доменные структуры, HTTP-handler);
       - во frontend (API-клиент/хук, компоненты, которые теперь используют реальные данные).
     - Убедись, что код компилируется:
       - go build ./... для backend;
       - npm run lint / npm run build или аналогичная команда для фронта.

Главные требования:
   - Не придумывай формат данных Столото из головы — опирайся на README.md и существующий код.
   - Не используй абстрактные заглушки, всё должно быть рабочим и согласованным.
   - Если находишь очевидные баги/несоответствия между README.md и кодом — аккуратно исправь и опиши, что именно ты привёл к единому контракту.
